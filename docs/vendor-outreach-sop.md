> 目标：在内容真实、有帮助的前提下，联系工具方争取转发、引用或资料补充。本文只提供外联流程和模板，不自动发送消息。

## 2. 标准外联模板

邮件 / 私信主题：

```text
[AIBoxPro 评测] {工具名} 在 {核心场景} 里的表现与适用人群整理
```

首次外联正文：

```text
你好，我们是 AIBoxPro，一个面向中文用户的 AI 工具决策平台。

我们最近整理了一篇关于 {工具名} 的内容：{页面标题}

TL;DR：
1. {一句话说明工具在什么场景适合谁}
2. {一句话说明我们如何呈现：对比 / 替代方案 / 场景指南}
3. {一句话说明对工具方或用户的价值}

完整链接：{URL}
转发素材包：{素材包链接或说明}

如果文中有事实信息需要修正，也欢迎直接指出；如果你们觉得对用户有帮助，也欢迎转发给需要选型的用户。没有强制合作诉求，只希望内容准确、对中文用户有用。
```

二次跟进模板（首次发送 5-7 天后）：

```text
你好，补充同步一下：上次发的 {页面标题} 已经有一些用户在看，我们也会持续更新价格、访问方式和适用场景。

如果你们有官方资料页、最新定价说明、使用案例或产品截图，可以发我们，我们会在核对后更新到页面里，并标注信息来源。

链接：{URL}
```

收到资料后的回复：

```text
收到，感谢补充。我们会按“官方资料 + 编辑核对”的口径更新，不会直接复制宣传文案。

更新完成后会把链接发你确认。
```

## 3. 转发素材包结构

每次外联前准备一个轻量素材包，建议放在团队云盘或 Notion，不放公开 Git 仓库。

```text
vendor-kit/{tool-or-page-slug}/
├── page-url.txt          # 站内页面链接
├── title.txt             # 推荐转发标题 2-3 个
├── summary.txt           # 80 字摘要 + 200 字摘要
├── key-points.md         # 3-5 个核心看点
├── images/
│   ├── cover.png         # 公众号 / 即刻封面
│   ├── xhs-cover.png     # 小红书封面
│   └── comparison.png    # 关键表格截图
└── disclosure.md         # 内容口径：doc-based / Lab / 更新时间
```

素材文案示例：

```text
80 字摘要：
AIBoxPro 整理了 {工具名} 在 {场景} 中的适用人群、国内访问方式、价格门槛和替代方案，适合正在选型的中文用户快速判断。

200 字摘要：
这篇内容不是泛泛介绍工具功能，而是围绕 {场景} 的真实选型问题，整理 {工具名} 与同类工具的差异、国内用户需要注意的访问和支付门槛，以及适合优先尝试的人群。页面会持续更新价格、可用性和相关对比链接。
```

| 即梦 AI | `/alternatives/midjourney` | 待填 | 官方入口 | 待填 | 待发送 | 否 | 否 | 否 | — | 待填 | 图像 / 视频素材 |

跟进节奏：

- 第 0 天：发送首次外联。
- 第 5-7 天：无回复则轻量跟进一次。
- 第 14 天：仍无回复则归档，不继续打扰。
- 对方指出事实错误时，优先修正，再考虑是否请求转发。

成功标准：

- 工具方确认或补充资料。
- 工具方在公众号、社群、官网、产品更新日志中引用或转发。
- 工具方愿意后续提供产品变更信息。
- 至少形成 5 个有效合作或转发案例，用于 Sprint 2 指标复盘。

## 2. 单篇内容分发 Checklist

适用内容：

- `/compare/*` 对比页
- `/alternatives/*` 替代方案页
- 未来 `/scenes/*` 场景指南页

发布前准备：

- [ ] 确认站内页面已上线，标题、摘要、结构化数据无明显错误。
- [ ] 准备 1 句 TL;DR：这篇内容解决谁的什么选择问题。
- [ ] 准备 1 张配图：对比表截图、工具卡片截图，或 I10 生成的封面图。
- [ ] 准备 3 个适合站外平台的标题变体，不同平台不要复制同一个标题。
- [ ] 在追踪表中新建一行，记录页面 URL、发布时间和负责人。

知乎：

- [ ] 搜索 1-2 个目标问题，例如“Cursor 和 Trae 哪个好用”“Claude Code 和 Cursor 区别”。
- [ ] 回答正文控制在站内原文 30% 以内，避免整篇搬运。
- [ ] 前 3 段直接给结论，中间放 3-5 个要点，文末自然附完整链接。
- [ ] 如果问题已有高赞回答，先补充差异化视角，不要重复堆表格。

V2EX / 即刻：

- [ ] 用 1 句结论开头，例如“如果你在国内写代码，Trae 的上手门槛比 Cursor 低，但 Cursor 的项目理解仍更强。”
- [ ] 放 1 张关键截图，不贴长文。
- [ ] 链接放在末尾，说明“完整对比在这里”。
- [ ] 评论区只回答真实问题，不主动私信用户。

稀土掘金 / SegmentFault：

- [ ] 适合 AI 编程、开发工具、GitHub Trending 相关内容。
- [ ] 可同步长文，但正文需要重写开头和结尾。
- [ ] 文末标注：`原文首发于 AIBoxPro，站内版本会持续更新。`
- [ ] 保留 2-3 个站内深链，例如工具详情页、相关对比页、替代方案页。

GitHub：

- [ ] 检查相关 awesome-list、工具仓库 Discussion、README 资源列表是否接受提交。
- [ ] 只在内容真正有帮助时提交，不在无关 issue 里贴链接。
- [ ] PR 描述说明链接价值，例如“中文用户的可用性与替代方案补充”。

微信公众号 / 小红书：

- [ ] 公众号使用完整长图文，正文末尾引导收藏和关注。
- [ ] 小红书使用 3-5 张卡片图，文案短句化，避免直接贴长链接。
- [ ] 小红书正文使用“主页查看完整对比”这类平台友好的表达。

## 3. 反模式清单

不要做：

- 不要在所有平台粘贴同一篇原文。
- 不要批量私信用户拉访问。
- 不要做小号矩阵、互赞、互评。
- 不要伪装成普通用户推荐 AIBoxPro。
- 不要在无关问题、无关仓库、无关帖子里硬贴链接。
- 不要夸大“实测”“最快”“最强”，除非页面有真实 Methodology Box。
- 不要搬运工具官网或其他目录站的大段原文。

应该做：

- 先回答问题，再给链接。
- 每个平台写适合该平台语气的摘要。
- 保留清楚的利益关系：这是 AIBoxPro 发布的整理或评测。
- 每次分发后记录结果，下一轮根据数据调整。

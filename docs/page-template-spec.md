# AIBoxPro 对比页与场景页模板规范

## 1. 文档目的

本文用于统一 AIBoxPro 两类核心 SEO / 决策页面的模板规范：

- 对比页
- 场景页

目标是让页面具备：

- 清晰的决策结构
- 一致的内容标准
- 可程序化生成
- 可人工精修

## 2. 总体原则

### 2.1 页面原则

- 页面必须帮助用户做选择
- 页面不能只做信息堆砌
- 页面不输出空泛判断
- 页面要把“适合谁”和“为什么”写清楚

### 2.2 语言原则

避免使用：

- 最强
- 第一
- 绝对最好
- 吊打
- 无脑选

建议使用：

- 更适合
- 更适合某类用户
- 如果你更看重 X，可以优先考虑 Y
- 对国内用户，建议先确认访问、账号和支付条件

### 2.3 生成原则

- 结构化字段负责事实
- AI 草稿负责成文
- 编辑负责最终结论和风险口径

## 3. 对比页模板规范

### 3.1 页面目标

承接高意图搜索词，例如：

- `Claude Code vs Codex`
- `Cursor vs Trae`
- `ChatGPT vs Kimi`

输出目标：

- 告诉用户两个工具的关键差异
- 按场景告诉用户谁更适合谁
- 给出国内使用提醒
- 引导用户继续查看详情页、场景页、替代页

### 3.2 URL 规范

建议：

```text
/compare/claude-code-vs-codex
/compare/cursor-vs-trae
```

### 3.3 SEO 规范

#### Title 模板

```text
Claude Code vs Codex：哪个更适合程序员？ | AIBoxPro
```

#### Description 模板

```text
AIBoxPro 从功能、价格、中文支持、国内使用情况和适合场景对比 Claude Code 与 Codex，帮你快速判断哪个更适合自己。
```

#### H1 模板

```text
Claude Code vs Codex：哪个更适合程序员？
```

### 3.4 页面结构

#### 模块 1：标题区

包含：

- H1
- 更新时间
- 相关标签

#### 模块 2：一句话结论

示例：

```text
Claude Code 更适合习惯终端开发的个人开发者，Codex 更适合已经深度使用 OpenAI 生态、希望把编码任务纳入协作流程的团队。
```

要求：

- 1 到 2 句话
- 必须是场景化结论
- 不输出绝对胜负

#### 模块 3：适合谁

分成两个区块：

- 更适合选 A 的人
- 更适合选 B 的人

每个区块建议 3 到 5 条。

#### 模块 4：核心区别

建议用 4 到 6 条差异点列出，例如：

- 产品形态
- 使用方式
- 任务深度
- 中文体验
- 国内使用门槛
- 团队协作方式

#### 模块 5：标准对比表

建议维度：

| 维度 | 工具 A | 工具 B |
| --- | --- | --- |
| 产品定位 |  |  |
| 适合人群 |  |  |
| 使用方式 |  |  |
| 中文支持 |  |  |
| 国内访问情况 |  |  |
| 价格模式 |  |  |
| 免费版 |  |  |
| API 支持 |  |  |
| 团队版 |  |  |
| 典型场景 |  |  |

#### 模块 6：价格与成本

要求：

- 写清 `免费 / Freemium / 订阅 / 按量付费`
- 不写模糊的“价格合理”
- 要标注：

```text
价格以官网为准，本站更新时间：YYYY-MM-DD
```

#### 模块 7：中文体验与国内使用提醒

要求：

- 单独成节
- 不给绕过方案
- 不给账号买卖建议

推荐模板：

```text
两款工具都可能受地区、网络、账号和支付条件影响。国内用户使用前，建议优先确认当前访问稳定性、注册门槛和付费方式。
```

#### 模块 8：典型使用场景

建议拆成：

- 如果你是个人开发者
- 如果你是小团队
- 如果你是企业用户

#### 模块 9：替代方案

示例：

- Claude Code 的替代方案
- Codex 的替代方案
- 国内可考虑的替代工具

#### 模块 10：FAQ

建议 3 到 5 个问题：

- 它们最大的区别是什么？
- 哪个更适合新手？
- 国内用户该先关注什么？

#### 模块 11：内链推荐

至少链接到：

- 两个工具的详情页
- 一个场景页
- 一个替代方案页

### 3.5 对比页内容长度建议

- 最小可发布版本：1200 到 1800 字
- 标准版本：1800 到 3000 字

### 3.6 对比页质量检查

发布前检查：

- 是否明确结论
- 是否写清适合谁
- 是否有国内提醒
- 是否有价格说明
- 是否有 FAQ
- 是否有内链

## 4. 场景页模板规范

### 4.1 页面目标

承接场景型搜索词，例如：

- `适合国内程序员的 AI 编程工具`
- `适合做 PPT 的 AI 工具`
- `适合团队代码审查的 AI 工具`

输出目标：

- 帮用户在一个任务场景里快速筛工具
- 给出推荐顺序和理由
- 让用户理解不同工具分别适合哪种需求

### 4.2 URL 规范

建议：

```text
/scenarios/best-ai-coding-tools-for-cn-developers
/scenarios/best-ai-tools-for-ppt
```

### 4.3 SEO 规范

#### Title 模板

```text
适合国内程序员的 AI 编程工具推荐（2026） | AIBoxPro
```

#### Description 模板

```text
AIBoxPro 从中文支持、国内访问、价格、协作能力和使用门槛出发，推荐适合国内程序员的 AI 编程工具。
```

#### H1 模板

```text
适合国内程序员的 AI 编程工具推荐
```

### 4.4 页面结构

#### 模块 1：标题区

包含：

- H1
- 更新时间
- 场景标签

#### 模块 2：一句话结论

示例：

```text
如果你是国内个人开发者，优先考虑上手快、中文支持更好的工具；如果你是团队负责人，应优先考虑协作性、成本控制和部署稳定性。
```

#### 模块 3：先给结论的推荐区

建议直接给 3 到 5 个推荐结果：

- 最推荐
- 最适合新手
- 最适合团队
- 最适合低成本尝试
- 最适合中文环境

#### 模块 4：推荐工具列表

每个工具卡片建议包含：

- 名称
- 一句话介绍
- 为什么推荐
- 适合谁
- 价格概览
- 国内使用提醒
- 查看详情 / 对比

#### 模块 5：对比表

建议列：

| 工具 | 适合谁 | 中文支持 | 国内可用性 | 价格 | 团队协作 | 推荐指数 |
| --- | --- | --- | --- | --- | --- | --- |

#### 模块 6：按需求细分推荐

建议拆成子段落：

- 如果你是个人开发者
- 如果你是团队负责人
- 如果你预算有限
- 如果你更看重中文体验
- 如果你更看重本地代码能力

#### 模块 7：不适合谁

这是很关键的差异化模块。

示例：

```text
如果你完全不熟悉命令行，某些终端型编码代理并不适合作为第一选择。
```

#### 模块 8：替代方案与延伸阅读

例如：

- Cursor 替代品
- Claude Code vs Codex
- 适合团队代码审查的 AI 工具

#### 模块 9：FAQ

建议问题：

- 新手应该从哪款开始？
- 国内用户要优先看什么？
- 团队采购时要注意什么？

### 4.5 场景页内容长度建议

- 最小可发布版本：1500 到 2200 字
- 标准版本：2200 到 3500 字

### 4.6 场景页质量检查

发布前检查：

- 是否先给推荐结论
- 是否明确不同用户的差异
- 是否包含工具对比表
- 是否有国内使用提醒
- 是否有 FAQ
- 是否链接详情页和对比页

## 5. 替代方案页简版模板

### 5.1 页面目标

解决：

- 原工具太贵
- 原工具不好接入
- 原工具不适合中文用户

### 5.2 页面结构

1. 什么情况下需要替代
2. 替代工具推荐
3. 各自适合谁
4. 国内适配提醒
5. 最终建议

## 6. CTA 规范

### 6.1 对比页 CTA

建议：

- 查看工具 A 详情
- 查看工具 B 详情
- 查看替代方案
- 查看相关场景页

### 6.2 场景页 CTA

建议：

- 查看详情
- 继续对比
- 收藏此页
- 提交工具

## 7. FAQ 生成规则

FAQ 应基于高频搜索意图，不要写成空话。

### 7.1 对比页 FAQ 类型

- 差异是什么
- 哪个更适合新手
- 哪个更适合团队
- 国内用户要注意什么

### 7.2 场景页 FAQ 类型

- 哪个最适合某类用户
- 是否有免费方案
- 团队采购要看什么
- 是否有国产替代

## 8. 页面生成的最小输入

### 8.1 对比页

至少需要：

- 两个工具的详情字段
- 两个工具的评分
- 两个工具的国内适配字段
- 一段编辑结论草稿

### 8.2 场景页

至少需要：

- 场景定义
- 至少 3 个推荐工具
- 推荐逻辑
- 国内使用提醒

## 10. 页面审核标准

### 审核通过最低标准

- 有清晰结论
- 有明确适合谁
- 有对比或推荐依据
- 有国内使用提醒
- 有更新时间
- 有内链

### 不建议发布的页面

- 只有简介没有结论
- 没有国内使用提醒
- 标题和正文不匹配
- 结论过度绝对
- 价格和官方信息明显过时

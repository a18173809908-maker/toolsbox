# AIBoxPro 一期任务拆解

本任务拆解对应 `whitepaper.md` 的第一期（第 1-30 天）：信任基础建设。

## 成功标准

打开任意核心工具详情页时，用户应能看到一致的 AIBoxPro 品牌、清晰的数据来源、最后更新时间、无乱码内容，以及可继续进入对比页的决策入口。

## 执行原则

- 先修复信任损耗，再扩内容规模。
- 先做静态入口和可验证字段，不急着做完整决策引擎。
- 每个任务都必须能用页面检查、构建检查或数据检查验证。

## P0：品牌与内容信任修复

### T1：统一品牌名

目标：全站用户可见品牌统一为 `AIBoxPro`，消除 `AiToolsBox` 混用。

范围：
- 页面标题、导航、页脚、metadata、结构化数据中的品牌名。
- 保留历史包名或内部变量名，除非它们会出现在用户界面或 SEO 输出中。

验证：
- 全仓搜索用户可见的 `AiToolsBox`。
- `npm run lint`
- `npm run build`

### T2：清理乱码资讯

目标：移除或隐藏会损害信任的乱码、过期资讯内容。

范围：
- 资讯展示入口。
- 资讯数据处理逻辑中产生乱码展示的路径。
- 不删除历史数据，除非确认是当前变更产生的测试数据。

验证：
- 首页和资讯相关模块不展示乱码。
- `npm run lint`
- `npm run build`

### T3：补齐基础合规页面

目标：上线最低可用的信任页面。

页面：
- 关于我们
- 隐私协议
- 工具提交说明
- 免责声明

验证：
- 页面可访问。
- 导航或页脚有合理入口。
- `npm run lint`
- `npm run build`

## P1：数据来源与时效标注

### T4：工具详情页增加数据来源与更新时间

目标：核心工具详情页展示数据来源、最后更新时间和待确认状态。

建议字段：
- `dataSource`
- `lastVerifiedAt`
- `pricingUpdatedAt`
- `availabilityUpdatedAt`
- `reviewStatus`

验证：
- 至少核心 AI 编程工具可展示更新时间。
- 过期或缺失状态不假装精确，显示“待确认”。
- `npm run lint`
- `npm run build`

### T5：数据时效性展示规则

目标：建立最小可用的过期标注逻辑。

第一期只需要覆盖：
- 定价超过 30 天：显示“价格待确认”
- 国内可用性超过 14 天：显示“状态待确认”
- 功能列表超过 60 天：显示“功能待更新”

验证：
- 构造缺失或过期字段时，页面显示降级文案。
- `npm run lint`
- `npm run build`

## P2：对比页模板

### T6：标准化对比页模板

目标：上线可承载 `Cursor vs Trae` 这类页面的基础模板。

首版模块：
- 标题与一句话结论
- 适合谁
- 核心区别
- 标准对比表
- 价格与成本
- 国内使用提醒
- Methodology Box 占位符
- FAQ

验证：
- 至少有一个静态或数据驱动样例页可访问。
- 页面 metadata 正确。
- `npm run lint`
- `npm run build`

## P3：首页决策入口静态框架

### T7：首页四大决策入口

目标：首页提供清晰决策路径，但不开发完整智能选型助手。

入口：
- 工具对比
- 场景搜索
- 寻找替代品
- 编辑榜单

验证：
- 首页入口可见，移动端不挤压或重叠。
- 入口可以先链接到占位页或现有列表页。
- `npm run lint`
- `npm run build`

## 建议执行顺序

1. T1：统一品牌名
2. T2：清理乱码资讯
3. T3：补齐基础合规页面
4. T4：工具详情页增加数据来源与更新时间
5. T5：数据时效性展示规则
6. T6：标准化对比页模板
7. T7：首页四大决策入口

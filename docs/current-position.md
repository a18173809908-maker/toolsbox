# AIBoxPro 当前定位

最后更新：2026-05-09

## 当前项目定义

AIBoxPro 不再只定位为“AI 工具导航站”。

当前更准确的定位是：

> 面向中文用户的 AI 工具选择、对比与使用决策平台。

当前长期方向以 `docs/aiboxpro_详细版竞品分析与发展路线方案.md` 为战略输入。本文档只负责把那份长方案压缩成日常执行时不容易跑偏的判断标准。

它服务的不是“我想随便看看有哪些 AI 工具”，而是更明确的决策链路：

```text
发现工具
-> 理解用途
-> 对比替代品
-> 判断国内可用性
-> 学会实际工作流
-> 为一个具体任务选出合适工具
```

工具库仍然是底层数据资产，但不再是唯一卖点。真正要拉开的差异，是帮助中文用户更快做选择。

## 当前强调什么

1. **中文用户决策信息**
   - 国内访问情况
   - 中文界面 / 中文支持
   - 人民币价格或本地支付限制
   - 注册门槛
   - 国产或国内更容易使用的替代品

2. **工具对比页**
   - 回答“我到底该用哪个”。
   - 能引用官方来源的地方优先引用官方来源。
   - 只有做过真实测试时，才发布画质、速度、稳定性等强结论。

3. **场景与教程内容**
   - 按用户要完成的任务组织内容，而不是只按工具名组织。
   - 重点场景包括 AI 视频生成、PPT、AI 写作、编程助手、数字人等。

4. **趋势追踪**
   - GitHub Trending 和 AI 资讯负责新鲜度和复访。
   - 趋势内容要尽量回连到工具、对比和场景，不做孤立资讯流。

## 目标产品结构

长期结构按详细方案推进，但分阶段落地：

- AI 工具库：作为底层数据资产，不再单独承担主要差异化。
- AI 资讯：提供新鲜度，但必须筛选和回连工具 / 场景。
- GitHub 趋势：吸引开发者和产品型用户，形成专业感。
- 工具教程：补足“怎么用”的深度。
- 工具榜单：面向明确问题做筛选，不做泛泛排名。
- 工具对比：回答“哪个更适合我”。
- 场景解决方案：面向任务组织内容，是后续主增长方向。
- 商业化推荐：等信任和内容资产建立后再推进。

执行顺序不是同时铺开全部模块，而是先用一个品类跑通可自动化链路：工具数据 -> 对比页 -> 替代品页 -> doc-based 场景指南 -> 自动内链。需要真人实测、人工审校、人工运营分发或厂商触达的内容暂不做。

## 当前主线

Sprint 3 的主线是 **AI 视频品类**。

AI 视频不是最终唯一方向，而是当前最适合验证详细方案的切入品类：国内外工具差异明显，用户决策成本高，也更需要“国内可用性、价格、注册、支付、生成效果、适合场景”的综合判断。

截至这次文档重置，已经完成：

- 视频工具补录和详情增强脚本。
- 12 个视频相关工具具备更完整详情数据。
- 10 篇视频对比页已通过脚本生成 / 发布。
- 对比页社交分发草稿生成能力已具备。
- 连通性数据层已经存在，支持真实测量数据。
- 首页、工具库、分类、资讯、趋势、对比、替代品和后台页面都已有基础实现。

接下来最重要的产品 / 内容工作：

1. 补强视频替代品页面，尤其是面向国内用户的 Runway 替代品。
2. 增加不依赖人工实测的 doc-based 视频场景页。
3. 把工具详情页、对比页、替代品页、场景页互相内链。
4. 对比内容继续绑定官方来源；没有实测的数据必须明确标注。

## 现在不做什么

- 不把“收录了多少工具”作为主卖点。
- 不批量生产很薄的工具摘要。
- 不在没有实测的情况下发布画质、速度、稳定性排名。
- 暂不做真人实测 Lab、人工深度评测、人工审校、人工运营分发、厂商触达。
- 暂不做登录、评论、社区评分、会员体系。
- 视频主线没跑通前，不快速扩张到太多新品类。

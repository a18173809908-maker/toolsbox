# AIBoxPro 分期路线图

来源：`docs/aiboxpro_详细版竞品分析与发展路线方案.md`

最后更新：2026-05-10

## 拆分原则

这份路线图把详细方案拆成多期执行任务。长期方向保持不变：

> 面向中文用户的 AI 工具选择、使用与趋势追踪平台。

执行上不一次性铺开所有模块，而是按“先跑通一个品类，再复制到更多场景”的方式推进：

```text
定位与数据底座
-> AI 视频样板品类
-> 内容矩阵扩展
-> 场景解决方案
-> 增长与分发
-> 商业化
```

当前执行约束：

> 所有需要人工参与的内容暂不做。

暂缓范围包括真人实测 Lab、人工深度评测、人工审校、手动运营分发、厂商触达、社区互动、线下访谈、人工维护的商业合作。当前优先做可工程化、可自动化、可由现有数据和官方来源支撑的内容与功能。

## Phase 0：定位与文档底座

状态：已完成（2026-05-09）。

目标：让 Codex、Claude Code 和后续协作者都不会被旧文档带偏。

任务：

- P0-1：保留 `aiboxpro_详细版竞品分析与发展路线方案.md` 作为长期战略源。
- P0-2：维护 `docs/current-position.md`，把长期方向压缩成日常执行判断。
- P0-3：维护 `CLAUDE.md` 和 `HANDOFF.md`，让 Claude Code 知道当前方向、入口文档和协作规则。
- P0-4：旧 Sprint、旧原型和临时审核材料统一归档，不再作为当前指令。
- P0-5：每次调整方向时同步更新 `docs/README.md`。

验收：

- `docs/README.md` 的“先读这里”指向清晰。
- Claude Code 只读 `CLAUDE.md` + `HANDOFF.md` 也能理解项目定位。
- 归档文件不会被误认为 active roadmap。
- `docs/manual-tasks.md` 只作为未来人工工作池，不作为近期路线图。

不做：

- 不继续维护多份互相冲突的 Sprint 草稿。

## Phase 1：数据与页面基础补齐

状态：进行中，约 80% 完成。已处理 AI 资讯、GitHub 趋势、信息源候选池、工具库排序与工具详情页决策信息。

目标：保证工具库、资讯、趋势、对比、替代品这些基础页面能承接后续内容。

任务：

- P1-1：工具库字段标准化，重点保留国内用户决策字段：`chinaAccess`、`chineseUi`、`priceCny`、`registerMethod`、`needsOverseasPhone`、`overseasPaymentOnly`。（已完成数据结构与页面展示，后续仍需做数据质量巡检）
- P1-2：工具详情页补齐“适合谁、怎么用、价格/权益、FAQ、替代品、相关对比”。（已补齐页面模块与自动决策提示，缺口主要是部分工具的数据完整度）
- P1-3：AI 资讯页保持来源、摘要、影响、适合关注人群和相关工具链接。（已完成）
- P1-3a：信息源采用候选池机制；自动发现/整理候选源，用户审核通过后才加入正式抓取。（已完成初版）
- P1-4：GitHub Trending 保持自动刷新和中文摘要。（已增强为摘要、场景、亮点、上榜原因、项目类型和适合人群）
- P1-5：对比页支持 JSON-LD、FAQ、官方来源、审核状态和 sitemap 收录。
- P1-6：替代品页具备专题能力，比如 `/alternatives/runway`。

验收：

- `npm run lint` 通过。
- `npm run build` 通过。
- 首页、工具库、工具详情、资讯、趋势、对比、替代品页面可访问。
- 新增内容能被 sitemap 收录。
- AI 资讯列表和详情页不再只有单调摘要；支持一句话摘要、关键信息、为什么重要、国内用户影响、适合人群和相关工具。
- 信息源候选可以在 `/admin/sources` 审核，通过后才进入正式 `sources`。
- 工具库按热度优先、时间其次排序；卡片展示适合场景、热度信号、免费额度、API / 开源等决策标签。
- 工具详情页展示“适合谁 / 需要谨慎”、国内用户实操信息、FAQ、国产替代、相关对比、相关资讯和同类工具。
- 新闻列表改为“爆点优先、时间其次、热度兜底”；右侧提供最近 3 天热门话题 Top 5。
- 新闻和 GitHub Trending 增加轻量分享能力：支持系统分享或复制摘要链接。

剩余缺口：

- 对现有工具做一轮数据完整度巡检，补齐缺失的 `howToUse`、`faqs`、`priceCny`、`registerMethod`、`cnAlternatives` 和教程链接。
- 对比页、替代品页和工具详情页的内链关系还需要按重点品类继续补强。
- sitemap 与重点页面收录状态需要在后续阶段统一复查。
- 暂不做登录态收藏、用户订阅、RSS 管理、站内深度分析 Key 配置和复杂个人中心；这些功能等内容流与热点机制稳定后再评估。

不做：

- 不新增登录、评论、会员或复杂后台权限。
- 不为了“工具数量”批量灌入低质量工具。

## Phase 2：AI 视频样板品类

状态：当前主线。

目标：用 AI 视频验证详细方案里的“选择效率 + 中文场景 + 工具对比 + 场景方案”模式，但当前只做不依赖人工实测和人工运营的部分。

任务：

- P2-1：完成视频工具池校验，确保可灵、即梦、海螺、Runway、Sora、Pika、Luma、Vidu、Hedra 等核心工具字段可信。
- P2-2：保留并自动审计 10 篇视频对比页，所有强判断必须有官方来源或标注“待实测”。
- P2-3：上线 Runway 替代品专题，面向国内用户解释可用性、价格、注册、支付和适用场景。
- P2-4：为视频品类补 3-5 篇 doc-based 场景页：文生视频、图生视频、短视频剪辑、数字人、产品展示视频。
- P2-5：把视频工具详情页、对比页、替代品页、场景页互相内链。
- P2-6：把 Lab 报告相关字段和页面能力预留好，但不发布真人实测报告。

验收：

- 至少 1 个视频替代品专题可访问并进 sitemap。
- 视频对比页不出现无依据的画质、速度、稳定性排名。
- 视频品类形成闭环：工具详情 -> 对比 -> 替代品 -> 场景页。
- 需要人工实测的 Lab 报告只保留为未来能力，不作为当前交付。

不做：

- 不写泛泛的“十大 AI 视频工具推荐”水文。
- 不扩展过多视频子分类，先用现有 `video` 大类。
- 暂不做真人实测 Lab。
- 暂不做人工审校作为交付前置条件。

## Phase 3：内容矩阵扩展

状态：Phase 2 跑通后启动。

目标：把 AI 视频样板复制到更多高价值品类。

建议顺序：

1. AI 写作 / 内容创作
2. AI PPT / 办公
3. AI 编程
4. AI 绘图
5. AI 自动化 / 企业提效

每个品类的标准任务：

- P3-x-1：补齐 8-15 个核心工具。
- P3-x-2：补齐国内用户决策字段。
- P3-x-3：发布 5-10 篇对比页。
- P3-x-4：发布 1 个替代品专题。
- P3-x-5：发布 3-5 篇场景方案页。
- P3-x-6：预留 Lab 或人工深度评测入口，但当前不把它作为交付要求。

验收：

- 每个品类都有工具库、对比、替代品、场景页四类内容。
- 每个品类至少有一个 doc-based 深度内容资产。
- 页面之间有明确内链，不形成孤岛页面。

不做：

- 不同时铺开多个新品类。
- 不用 AI 自动生成未经审核的大量薄页面。
- 暂不做人工深度评测。

## Phase 4：场景解决方案主线

状态：Phase 2/3 有稳定内容资产后启动。

目标：把网站从“工具选择平台”进一步推到“具体任务解决方案平台”。

优先场景：

- 学生：AI 写论文、AI 英文作文、AI 翻译长文档、AI 思维导图。
- 创作者：小红书笔记、公众号文章、短视频脚本、AI 视频生成。
- 开发者：AI 编程助手、开源 AI 项目、副业项目拆解。
- 中小团队：客服自动化、办公自动化、内容批量生产、数据分析。

任务：

- P4-1：设计 `scenes` 数据结构或等价内容模型。
- P4-2：新增 `/scenes` 列表页和 `/scenes/[slug]` 详情页。
- P4-3：场景页支持关联工具、相关对比、FAQ、HowTo / Article JSON-LD。
- P4-4：先发布 10 篇 doc-based 场景方案，每篇必须有明确目标用户、步骤、工具组合和替代方案；实操截图和真人案例暂缓。
- P4-5：工具详情页反向展示“相关场景指南”。

验收：

- `/scenes` 和至少 10 个场景详情页可访问。
- 每篇场景页至少关联 3 个工具和 1 篇对比或替代品页。
- sitemap 收录场景页。
- 首页和导航给场景指南明显入口。

不做：

- 不做泛泛教程集合。
- 不写没有实操路径的场景文章。
- 暂不要求人工截图、真人案例或手动体验记录。

## Phase 5：增长、运营与分发

状态：暂缓。当前不做需要人工运营参与的事项。

目标：让内容被看见，并用数据决定下一个品类。

任务：

- P5-1：保留小红书、公众号、知乎分发稿生成能力，但暂不执行人工发布。
- P5-2：建立可自动化的每周内容候选池：趋势更新、对比更新、场景更新。
- P5-3：用 Search Console / Vercel Analytics / 数据库统计观察 SEO 入口词和页面转化。
- P5-4：把高频搜索词回填到工具、对比、榜单和场景选题。
- P5-5：厂商触达和社区分发 SOP 保留文档，不作为当前执行任务。

验收：

- 每周至少自动生成或更新 3 个内容候选 / 页面资产。
- 每个重点页面可生成外部分发素材，但不要求人工发布。
- 能列出下一阶段优先品类的真实数据依据。

不做：

- 不为了日更牺牲事实准确性。
- 不在没有数据反馈前频繁改战略方向。
- 暂不做人工分发、社群运营、厂商触达。

## Phase 6：商业化验证

状态：暂缓，等内容信任和稳定流量后启动。

目标：验证收入模式，但不破坏用户信任。

可选方向：

- 工具商广告位。
- 付费收录，但必须明确标识。
- 联盟返佣。
- 付费内容产品，如工具选型清单、AI 副业教程、自动化模板。
- 企业工具选型咨询。

启动条件：

- 至少 3 个品类形成内容闭环。
- 至少 10 篇高质量对比 / 场景页有稳定自然流量。
- 已有基础数据能证明用户在做工具选择。
- 用户确认可以投入人工商务或运营参与。

验收：

- 商业化内容和编辑内容有清楚区分。
- 推荐理由仍然围绕用户决策价值。
- 不因为商业化改变未实测结论。

不做：

- 不过早做复杂会员系统。
- 不隐藏广告或付费推荐属性。

## 当前推荐执行顺序

1. 先与用户讨论并优化 Phase 1 的任务边界。
2. 用户确认后，完成 Phase 1 的数据与页面基础补齐。
3. 继续 Phase 2：AI 视频替代品专题 + doc-based 场景页 + 自动内链。
4. 暂缓 Lab 报告、人工审校、人工运营分发和厂商触达。
5. Phase 2 的自动化闭环跑通后，再进入 Phase 4 的场景页模型。
6. 用可自动获取的数据反馈决定 Phase 3 下一个扩展品类。
7. Phase 6 商业化等自然流量、可信内容资产和人工投入条件都具备后再做。

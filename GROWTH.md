# AiToolsBox — 增长与内容运营设计文档

> 站在运营者 + 用户粘性双重视角，梳理当前问题、差异化定位、内容策略与功能优先级。
> 本文是战略文档，不是开发任务单。每个功能在动工前应先在此处更新细节设计。

---

## 一、现状诊断

### 用户流失漏斗

```
用户到达首页
    ↓ 看工具列表
    ↓ 点进工具详情
    ↓ 跳转外部工具网站   ← 流失
    
    ↓ 看 GitHub Trending
    ↓ 点进仓库详情
    ↓ 跳转 GitHub        ← 流失
    
    ↓ 看 AI 资讯
    ↓ 点进资讯详情
    ↓ 跳转原文           ← 流失
```

**核心问题：站点是跳板，不是目的地。** 用户每次到来都以离开告终，没有任何留下来的理由，也没有任何个人化的东西让他感觉"这是我的地方"。

### 内容价值不足的具体表现

| 页面 | 现状 | 缺什么 |
|---|---|---|
| 工具详情 | 一段英文 + 一段中文 | 没有定价细节、没有用户评分、没有使用场景、没有替代品 |
| 工具列表 | 按收录时间排序 | 没有热度、没有评分、没有"国内可用"标记 |
| GitHub Trending | 标题 + 描述 | 没有为什么今天突然火、没有历史趋势 |
| AI 资讯 | 标题 + 摘要 | 没有编辑点评、没有关联工具 |
| 分类页 | 工具列表 | 没有分类介绍、没有推荐理由、没有场景引导 |

## 二、差异化定位

### 市场现状

目前主要竞品：
- **There's An AI For That** — 工具数量最多，但无中文，无 GitHub，无资讯
- **FutureTools** — 有分类筛选，但同样无中文视角
- **AI 研习社 / 量子位工具库** — 中文内容，但更新慢，无 GitHub 趋势

### 我们的差异化定位

> **中国用户视角的 AI 全景站**
>
> 不只是工具目录，而是帮中国用户回答三个问题：
> 1. 这个工具**国内能用吗**，有没有中文界面？
> 2. 同类工具中**哪个最适合我**，有什么免费替代？
> 3. AI 领域**现在最值得关注**的是什么（工具 + 开源 + 资讯）？

这个定位目前市场上没有人做好。它天然形成护城河，因为：
- 「国内可用」信息需要持续维护，新进入者很难快速追上
- 中文 AI 内容 + 开源趋势的组合是独特的
- 中国 AI 用户基数大，但优质中文导航站极少

## 三、用户粘性设计

### 3.1 回访钩子分类

| 类型 | 机制 | 频率 |
|---|---|---|
| **实时更新** | GitHub Trending 每小时刷新，醒目展示「X 分钟前更新」 | 每小时 |
| **每日内容** | 「今日精选」—— 编辑每天推荐一个工具，附短评 | 每天 |
| **每周内容** | AI 周报邮件 —— 本周工具/仓库/资讯精华 | 每周一 |
| **事件驱动** | 工具有重大更新时推送通知（邮件/浏览器通知） | 不定期 |

### 3.2 个人化：My AI Stack（我的工具箱）

**不需要注册**，基于 localStorage + URL hash 实现。

用户流程：
1. 在工具卡片上点「+ 加入工具箱」
2. 首页右上角出现「我的工具箱 (3)」入口
3. `/stack` 页面展示已收藏工具，可以拖拽排序、加备注
4. 点「分享」生成 `/stack/abc123` 短链，他人可查看
5. 二维码或复制链接分享到社群

**为什么这个很重要：**
- 分享链接 = 免费自传播，每个分享都带来新用户
- 「我建了一套 AI 工具组合」是有分享欲的内容
- 不需要注册降低了参与门槛

### 3.3 社区信号：轻量评分

不做复杂评论系统，只做最轻的「有用/没用」：

- 每个工具页：👍 有用 / 👎 踩（无需登录，按 IP 限流）
- 聚合成「社区评分」展示在工具卡片上
- 工具列表支持「按好评率排序」
- 评分数据形成竞争壁垒（越积越有价值）

### 3.4 邮件订阅

**这是最重要的运营资产。** PV 是租来的（依赖搜索引擎），邮箱列表是买来的。

AI 周报内容结构（每周一发送）：
```
📌 本周 5 个值得关注的 AI 工具
  - 工具名 + 一句话介绍 + 编辑短评

🔥 GitHub 本周最火仓库 Top 3
  - 仓库 + 为什么突然火

📰 AI 行业重要资讯 3 条
  - 标题 + 编辑观点

🆕 本周新收录工具
```

技术实现：Resend（简单、便宜，有中文文档）

## 四、SEO 内容矩阵

### 4.1 高价值长尾页面（可自动生成）

#### A. 场景落地页 `/for/[use-case]`

用户真实搜索意图：「写代码用什么 AI」「做设计用什么 AI」「剪视频用什么 AI」

页面结构：
- H1: 「写代码最好用的 AI 工具（2026 最新）」
- 编辑推荐 Top 3（带理由）
- 完整工具列表（按评分排序）
- 常见问题 FAQ（结构化数据，有利于 AI 搜索结果）
- 相关场景推荐

预计场景页：`coding` / `writing` / `design` / `video` / `audio` / `research` / `productivity` / `marketing`

#### B. 替代品页 `/alternatives/[tool]`

用户搜索意图：「Midjourney 替代品」「ChatGPT 免费替代」「Notion AI 替代」

这类词搜索量大、购买意图强，是目录站最高价值的 SEO 词。

页面结构：
- H1: 「Midjourney 的 5 个替代品（免费/付费）」
- 替代品列表 + 对比维度（价格/功能/国内可用）
- 推荐理由（1-2 句编辑点评）

#### C. 对比页 `/compare/[a]-vs-[b]`

用户搜索意图：「ChatGPT vs Claude」「Midjourney vs Stable Diffusion」

页面结构：
- 并排对比表格（价格/功能/适用场景/评分）
- 各维度详细说明
- 编辑结论：谁更适合哪类用户

#### D. 标签页 `/tag/[tag]`

- `/tag/free` — 免费 AI 工具
- `/tag/china-accessible` — 国内可直接访问
- `/tag/open-source` — 开源
- `/tag/api-available` — 提供 API
- `/tag/chinese-ui` — 有中文界面
- `/tag/no-signup` — 无需注册

### 4.2 编辑内容（需要人工投入）

| 内容类型 | 频率 | SEO 价值 | 运营成本 |
|---|---|---|---|
| 工具深度评测 | 每周 2-3 篇 | ⭐⭐⭐ | 高 |
| AI 周报（博客版） | 每周 1 篇 | ⭐⭐ | 中 |
| 专题合集 | 每月 2-3 个 | ⭐⭐⭐ | 中 |
| 工具更新追踪 | 实时 | ⭐⭐ | 低（自动化） |

## 五、「国内可用」信息体系

这是最核心的差异化，需要建立完整的信息体系。

### 5.1 工具可用性标签

每个工具需要补充以下字段：

```
accessibility: {
  china: "blocked" | "accessible" | "vpn-required" | "unknown"
  chineseUI: boolean
  freeQuota: string | null   // e.g. "每天 10 次免费"
  apiAvailable: boolean
  openSource: boolean
  selfHostable: boolean
}
```

### 5.2 显示方式

工具卡片角落加小标签：
- 🟢 国内可用
- 🟡 需要 VPN
- 🔴 国内无法访问
- 🇨🇳 中文界面
- ∞ 永久免费额度
- `< >` 开源

### 5.3 维护机制

- 初始数据：人工录入
- 更新机制：用户举报「信息有误」→ 人工复核
- 自动检测：每周 HEAD 请求检测工具网址是否可达（区分国内 IP）

## 七、功能优先级矩阵

### 评分标准
- **影响力**：对用户留存/获客的影响程度
- **可行性**：当前技术栈实现难度
- **紧迫性**：是否影响核心体验

### 优先级排序

| 功能 | 影响力 | 可行性 | 紧迫性 | 优先级 |
|---|---|---|---|---|
| 工具可用性标签（国内/VPN/免费） | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ | **P0** |
| 场景落地页 `/for/[use-case]` | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | **P0** |
| 邮件订阅入口 + 周报 | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ | **P1** |
| My AI Stack（我的工具箱）| ⭐⭐⭐ | ⭐⭐ | ⭐ | **P1** |
| 替代品页 `/alternatives/[tool]` | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ | **P1** |
| 工具👍/👎 评分 | ⭐⭐ | ⭐⭐⭐ | ⭐ | **P2** |
| 今日精选（编辑推荐） | ⭐⭐ | ⭐⭐⭐ | ⭐ | **P2** |
| 对比页 `/compare/[a]-vs-[b]` | ⭐⭐⭐ | ⭐ | ⭐ | **P2** |
| 标签页 `/tag/[tag]` | ⭐⭐ | ⭐⭐⭐ | ⭐⭐ | **P2** |
| 工具提交表单（用户投稿） | ⭐⭐ | ⭐⭐ | ⭐ | **P3** |
| 工具更新追踪（Changelog）| ⭐⭐ | ⭐ | ⭐ | **P3** |
| 用户账号系统 | ⭐⭐ | ⭐ | ⭐ | **P3** |

### P0 详细说明

#### P0-1：工具可用性标签
**为什么最优先：** 这是最核心的差异化，中国用户最关心的信息，其他站不做，一旦建立难以追赶。
**最小实现：** schema 加字段 → 工具卡片/详情页显示标签 → 人工录入初始数据。

#### P0-2：场景落地页
**为什么最优先：** 纯 SEO 收益，内容可半自动生成，能快速带来精准流量。「写代码用什么 AI」这类词月搜索量大、竞争相对低。
**最小实现：** 建 8 个核心场景页，每个页面：标题 + 编辑推荐 3 个 + 完整工具列表。

## 九、技术预留

以下功能在设计 schema 时需要预留字段，避免后期大规模迁移：

```sql

ALTER TABLE tools ADD COLUMN free_quota text;             -- 免费额度说明
ALTER TABLE tools ADD COLUMN api_available boolean;       -- 是否提供 API
ALTER TABLE tools ADD COLUMN open_source boolean;         -- 是否开源
ALTER TABLE tools ADD COLUMN github_repo text;            -- 对应 GitHub 仓库
ALTER TABLE tools ADD COLUMN alternatives text[];         -- 替代品 tool id 列表
ALTER TABLE tools ADD COLUMN use_cases text[];            -- 使用场景标签
ALTER TABLE tools ADD COLUMN upvotes integer DEFAULT 0;   -- 👍 数
ALTER TABLE tools ADD COLUMN downvotes integer DEFAULT 0; -- 👎 数

-- 新增表
CREATE TABLE subscriptions (                              -- 邮件订阅
  email text PRIMARY KEY,
  created_at timestamp DEFAULT now(),

  tool_id text, clicked_at timestamp, referrer text
```

## 十、国内用户定位的专项要求

> 核心用户群为中国大陆用户，以下要求从一开始就要纳入设计，后期改造成本极高。

### 10.1 SEO：百度优先

百度与 Google 的抓取机制差异较大：

| 项目 | 要求 |
|---|---|
| **渲染方式** | 百度蜘蛛对 JS 渲染支持弱，所有核心内容必须 SSR/SSG 输出，不能依赖客户端渲染 |
| **Sitemap** | 需额外提交百度站长平台，格式略有差异 |
| **页面速度** | 百度 MIP（移动加速页）可考虑，首屏 < 2s |
| **内链结构** | 百度权重对内链更敏感，场景页 → 工具页 → 分类页要形成完整内链网络 |
| **关键词** | 以中文关键词为主："免费 AI 绘图""国内可用 AI 工具"等 |
| **百度统计** | GA4 在国内加载慢且时常被屏蔽，主数据源改为百度统计，GA4 作为辅助 |

### 10.2 流量渠道：微信生态为核心

国内用户获客渠道优先级：

1. **百度搜索** — 长尾词自然流量，核心 SEO 阵地
2. **微信公众号** — AI 周报以公众号图文为主载体，邮件为辅
3. **知乎** — AI 工具推荐类问题回答，引流效率高
4. **小红书** — 工具测评类内容，用户决策场景强
5. **抖音/视频号** — 工具演示短视频，中期布局

**技术影响：**
- OG 图要同时符合微信分享卡片规范（800×400 最优）
- 微信内打开要处理 UA 检测，确保样式正常
- 考虑接入微信登录（Phase 2 用户系统）

### 10.3 备案要求（变现硬门槛）

百度联盟、阿里妈妈等国内广告联盟要求网站完成 ICP 备案。

**备案路径：**
- 需要：国内服务器（阿里云/腾讯云）+ 已注册域名 + 企业/个人身份信息
- 当前 Vercel 托管的境外服务器**无法直接备案**
- **方案**：
  - 短期：Vercel 继续用于开发/测试，正式上线前迁移
  - 推荐方案：腾讯云 Lighthouse（香港节点）过渡 → 国内节点备案
  - 域名：提前注册 `.cn` 或 `.com.cn` 域名，备案更顺畅

**备案时间：** 一般 7–20 个工作日，需提前规划，不能临时抱佛脚。

## 十一、百度联盟变现规划

### 11.1 接入条件

| 条件 | 状态 | 备注 |
|---|---|---|
| ICP 备案 | ❌ 待办 | 硬门槛，见 10.3 |
| 日 IP ≥ 300 | ❌ 待达成 | 流量达标后申请 |
| 内容健康合规 | ✅ | 工具导航类内容风险低 |
| 网站运营 ≥ 3 个月 | ❌ 待达成 | 建议尽早上线开始计时 |

### 11.2 广告位规划

**原则：广告位在布局设计时就预留好位置，避免后期插入破坏页面结构。**

| 位置 | 类型 | 尺寸 | 优先级 |
|---|---|---|---|
| 工具列表第 5/10 条间 | 原生信息流广告 | 与工具卡同宽 | P0 |
| 工具详情页右侧栏 | 展示广告 | 300×250 | P0 |
| 资讯列表间插（每 8 条） | 原生广告 | 全宽 | P1 |
| 页面底部 Footer 上方 | 横幅广告 | 728×90 | P1 |
| 分类页顶部 | 展示广告 | 全宽横幅 | P2 |

**实现方式：**
- 广告位用 `<AdSlot />` 组件占位，传入 `slotId` prop
- 未接入联盟时渲染为空（不影响布局），接入后注入脚本
- 广告脚本懒加载（`loading="lazy"`），避免影响 LCP 和 Lighthouse 分数

### 11.3 其他变现方式（互补）

百度联盟是基础收入，同时可布局：

| 方式 | 收入模型 | 适合阶段 |
|---|---|---|
| **付费推荐位** | 工具厂商付费排到分类第一 | 日 UV > 1000 |
| **工具测评合作** | 工具厂商付费出测评文章 | 内容有影响力后 |
| **联盟 CPS** | 工具订阅转化佣金（部分工具有） | 任何阶段 |
| **知识星球/会员** | 高质量 AI 工具使用指南 | 有稳定用户群后 |

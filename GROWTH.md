## 二、现状诊断

### 用户流失漏斗

```
用户到达首页
    ↓ 看工具列表
    ↓ 点进工具详情       ← 页面单调，信息量不足
    ↓ 跳转外部工具网站   ← 流失

    ↓ 看 GitHub Trending
    ↓ 点进仓库详情       ← 页面单调，没有 README/Topics
    ↓ 跳转 GitHub        ← 流失

    ↓ 看 AI 资讯
    ↓ 点进资讯详情       ← 无编辑点评，无关联工具
    ↓ 跳转原文           ← 流失
```

**核心问题：站点是跳板，不是目的地。** 三次访问都以跳出告终，没有留下来的理由，也没有任何个人化内容。

### 各页面现状与缺口

| 页面 | 现状 | 核心缺口 |
|---|---|---|
| 工具详情 | 一段英文 + 一段中文 + 相关工具 | 无官网 URL、无国内可用标注、无功能亮点列表、无定价细节 |
| GitHub 仓库详情 | 描述 + Stars 统计 | 无 README、无 Topics、无语言分布，内容过于单薄 |
| 工具列表（首页） | 按收录时间排序 | 无热度排序、无国内可用标记、无评分 |
| AI 资讯 | 标题 + 摘要 | 无编辑点评、无关联工具推荐 |
| 分类页 | 工具列表 | 无分类介绍、无使用场景引导、无精选推荐 |
| **导航栏** | 仅「首页」「AI 资讯」 | 缺 GitHub 趋势、工具库、分类等入口 |

## 三、差异化定位

### 竞品分析

| 竞品 | 优势 | 劣势 |
|---|---|---|
| There's An AI For That | 工具数量最多 | 无中文、无 GitHub、无资讯 |
| FutureTools | 分类筛选好 | 无中文视角、更新慢 |
| AI 研习社 / 量子位工具库 | 中文内容 | 更新慢、无 GitHub 趋势、无可用性标注 |

### 我们的定位

> **中国用户视角的 AI 全景导航站**
>
> 帮中国用户回答三个问题：
> 1. 这个工具**国内能用吗**？有中文界面吗？有免费额度吗？
> 2. 同类工具中**哪个最适合我**？有什么平替？
> 3. AI 领域**现在最值得关注**的是什么（工具 + 开源 + 资讯）？

护城河来源：
- 「国内可用」信息需持续维护，后来者难以快速追上
- 工具 + GitHub 开源 + 资讯三合一，组合独特
- 中文 AI 用户基数大，优质导航站极少

## 四、导航栏设计

### 现状问题

现在导航只有「首页 Home」「AI 资讯 News」，用户无法快速找到 GitHub 趋势和工具库入口，站内结构对用户不透明。

**SEO 影响：** 导航链接是搜索引擎判断站点结构的重要信号，完整的导航有助于各频道页面获得更多权重。

### 目标导航结构

```
[Logo · AiToolsBox]   首页   AI 工具库   GitHub 趋势   AI 资讯   [⌘K 搜索]   [订阅周报]
```

| 导航项 | 链接 | 说明 |
|---|---|---|
| 首页 | `/` | 综合聚合页，保持现状 |
| AI 工具库 | `/tools` 或锚点跳首页工具区 | 独立的工具列表页，支持筛选排序 |
| GitHub 趋势 | `/trending` | 需新建趋势汇总页（现在只有详情页） |
| AI 资讯 | `/news` | 已有 |
| ⌘K | 弹窗 | 已有，接了真实搜索 API |
| 订阅周报 | 弹窗/跳锚点 | 邮件订阅入口，运营私域核心 |

### 实现注意事项

- 活跃状态判断不能用 `window.location.pathname`（SSR 会 hydration 报错），改用 Next.js 的 `usePathname()` hook
- 导航为 `<nav>` 语义标签，链接用 `<Link>` 组件，保证 SEO 可抓取
- 移动端折叠为汉堡菜单，导航项不超过 5 个可见

## 五、工具详情页设计

### 问题根源

**数据贫困导致页面单调。** 目前工具表只有：名字、描述（中/英）、分类、定价标签。缺少：官网 URL（最关键）、功能特点、定价细节、国内可用性、截图。没有数据，UI 再好看也是空架子。

### 解决路径：先补数据，再升级 UI

#### Step 1 — Schema 扩充（优先）

```sql
ALTER TABLE tools ADD COLUMN url text;
-- 工具官网地址，「访问官网」按钮的目标

ALTER TABLE tools ADD COLUMN china_access text DEFAULT 'unknown';
-- 'accessible'（国内直连）| 'vpn-required'（需VPN）| 'blocked'（无法访问）| 'unknown'

ALTER TABLE tools ADD COLUMN chinese_ui boolean DEFAULT false;
-- 是否有中文界面

ALTER TABLE tools ADD COLUMN free_quota text;
-- 免费额度说明，如「每天 50 次」「永久免费基础版」

ALTER TABLE tools ADD COLUMN api_available boolean DEFAULT false;
-- 是否提供 API

ALTER TABLE tools ADD COLUMN open_source boolean DEFAULT false;
-- 是否开源

ALTER TABLE tools ADD COLUMN github_repo text;
-- 对应的 GitHub 仓库（如 openai/whisper），可与 trending 数据联动

ALTER TABLE tools ADD COLUMN features text[];
-- 核心功能点列表，如 ['支持中文', 'API可用', '无需注册']

ALTER TABLE tools ADD COLUMN pricing_detail text;
-- 定价细节说明，如「免费版每天50次，Pro $20/月无限次」

ALTER TABLE tools ADD COLUMN alternatives text[];
-- 替代品工具 ID 列表，用于替代品页和详情页推荐

ALTER TABLE tools ADD COLUMN upvotes integer DEFAULT 0;
ALTER TABLE tools ADD COLUMN downvotes integer DEFAULT 0;
```

## 六、GitHub 仓库详情页设计

### 问题根源

数据源没有被充分利用。GitHub 公开 API（无需 Token，公开仓库每 IP 每小时 60 次请求）可以免费获取：
- README 全文（Markdown → HTML 渲染）
- Topics 标签
- 语言分布
- 贡献者列表
- 最近更新时间

这些内容服务端渲染后，页面内容量提升 10 倍，SEO 价值极大。

### 用户视角

用户进 GitHub 仓库详情页，核心问题是：
1. 这个仓库是干什么的？（README）
2. 它为什么今天突然火？（Stars 增量 + 来自哪里）
3. 我能用吗 / 怎么用？（部署难度、语言）

### 运营视角

- 每个热门仓库对应大量用户搜索（仓库名、作者名），详情页可以截获这些搜索流量
- README 内容 = 免费的 SEO 文字，抓取后完全合法
- 如果仓库对应某个 AI 工具，联动推荐该工具 → 工具详情页内链

### 页面布局

```
┌──────────────────────────────────────────────────────────┐
│  [owner 头像 80px]   owner / 仓库名（大号）               │
│  [语言色点] Rust   ★ 49,500   今日 +8,399  🔥           │
│                                                          │
│  [在 GitHub 查看 ↗]（深色主按钮）                        │
│  [复制克隆命令]  [复制 npm 安装命令（如有）]              │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│  今日为何上榜？（编辑/AI 自动生成的一段解读）              │
│  「Warp 今日新增 +8,399 Stars，原因是...」                │
└──────────────────────────────────────────────────────────┘

┌───────────────────┬──────────────────────────────────────┐
│  数据面板          │  README（从 GitHub API 实时获取，     │
│                   │  Markdown → HTML 渲染，SSR 输出）      │
│  今日  +8,399     │                                       │
│  本周  +12,000    │  ## What is Warp?                     │
│  本月  +35,000    │  Warp is a modern, Rust-based...      │
│                   │                                       │
│  Topics:          │  ## Features                          │
│  #ai  #terminal   │  - AI-powered autocomplete            │
│  #rust            │  - Collaborative sessions             │
│                   │  ...                                  │
│  语言分布:         │                                       │
│  ██████ Rust 94%  │                                       │
│  ██ Shell 6%      │                                       │
│                   │                                       │
│  [广告位 300×250] │                                       │
└───────────────────┴──────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│  相关 AI 工具推荐（如果仓库对应某个工具，联动展示）         │
│  「Warp 有对应的商业版，查看工具详情 →」                   │
└──────────────────────────────────────────────────────────┘
```

### SEO 考量

- `<title>`: `{仓库名} — GitHub {语言} 开源项目 今日 +{gained} Stars`
- 内容完全 SSR，README 在服务端渲染为 HTML
- 每次请求动态生成（`revalidate = 3600`），内容保持新鲜
- 内链到相关 AI 工具详情页

### 技术实现要点

- 调 `https://api.github.com/repos/{owner}/{repo}` 获取基本信息
- 调 `https://api.github.com/repos/{owner}/{repo}/readme` 获取 README（Base64 解码后用 `marked` 渲染）
- 速率限制：匿名 60 次/小时，如遇限制可加 GitHub Token（`GITHUB_TOKEN` 环境变量）
- README 中的图片路径需要处理（相对路径转绝对路径）
- 危险 HTML 过滤（`DOMPurify` 或 `sanitize-html`）

## 七、用户粘性设计

### 7.1 回访钩子

| 类型 | 机制 | 触发频率 |
|---|---|---|
| 实时更新 | GitHub Trending 每小时刷新，首页醒目展示「X 分钟前更新」 | 每小时 |
| 每日内容 | 「今日精选」—— 编辑每天推荐 1 个工具，附 100 字短评 | 每天 |
| 每周内容 | AI 周报（公众号 + 邮件双渠道）—— 本周工具/仓库/资讯精华 | 每周一 |
| 事件驱动 | 工具重大更新、仓库爆火时推送 | 不定期 |

### 7.2 My AI Stack（我的工具箱）

**不需要注册**，基于 `localStorage` + 可分享的 URL hash 实现。

用户流程：
1. 工具卡片右上角点「+ 加入我的工具箱」
2. 首页导航出现「我的工具箱 (3)」入口
3. `/stack` 页展示已保存工具，可拖排序、加备注
4. 点「生成分享链接」→ `/stack/[hash]` 他人可查看（只读）

**传播机制：** 「我的 AI 工具组合」是有分享欲的内容，一个链接带来多个新用户，无成本裂变。

**SEO 影响：** `/stack/[hash]` 页面设为 `noindex`，不影响 SEO。

### 7.3 轻量评分（👍/👎）

- 工具页顶部：「这个工具对你有帮助吗？」+ 👍 有用 / 👎 没用
- 按 IP + 工具 ID 限流（24小时内只能投一次）
- 聚合为「社区评分」显示在工具卡片
- 工具列表新增「按好评率排序」维度
- 评分数据同时作为排序信号，好的工具自然排前面

**SEO 影响：** 投票用 API 路由处理，不影响页面结构。评分数据写入 JSON-LD 的 `AggregateRating`，有利于 SEO。

### 7.4 邮件 / 公众号订阅

**这是最重要的运营资产。** 搜索流量是租来的，邮件列表和公众号粉丝是自己的。

AI 周报内容结构（每周一）：
```
📌 本周 5 个值得关注的 AI 工具  + 编辑短评
🔥 GitHub 本周最火仓库 Top 3   + 为什么火
📰 AI 行业重要资讯 3 条        + 编辑观点
🆕 本周新收录                  + 简介
```

实现：邮件用 Resend，公众号图文为主载体，邮件为辅。订阅入口放首页 Hero 下方（显眼位置）+ 文章页底部。

## 八、SEO 内容矩阵

**原则：所有 SEO 页面必须 SSR/SSG 输出，核心内容不依赖客户端渲染（百度蜘蛛对 JS 渲染支持弱）。**

### 8.1 场景落地页 `/for/[use-case]`

搜索意图：「写代码用什么 AI」「AI 视频生成工具」「免费 AI 绘图」

页面结构：
- `<h1>`: 「写代码最好用的 AI 工具（2026 最新）」
- 编辑推荐 Top 3（带推荐理由）
- 完整工具列表（按评分/热度排序）
- FAQ 结构化数据（Google/百度精选摘要友好）
- 相关场景横向导航

核心场景（优先）：
`coding` / `writing` / `image` / `video` / `audio` / `research` / `productivity` / `marketing`

### 8.2 替代品页 `/alternatives/[tool]`

搜索意图：「Midjourney 替代品」「ChatGPT 免费替代」— 购买决策场景，转化率高。

页面结构：
- `<h1>`: 「{工具名} 的 5 个替代品（含免费选项）」
- 对比表格：功能 / 价格 / 国内可用 / 评分
- 编辑推荐结论
- 面包屑 + 内链到各工具详情

### 8.3 对比页 `/compare/[a]-vs-[b]`

搜索意图：「ChatGPT vs Claude」「Midjourney vs Stable Diffusion」

页面结构：
- `<h1>`: 「{A} vs {B}：哪个更适合你？（2026 详细对比）」
- 并排对比表（价格/功能/适用场景/国内可用/评分）
- 各维度详细分析
- 编辑结论：适合哪类用户

### 8.4 标签页 `/tag/[tag]`

| 标签 | 路由 | 目标关键词 |
|---|---|---|
| 国内可用 | `/tag/china-accessible` | 「国内可用 AI 工具」 |
| 免费 | `/tag/free` | 「免费 AI 工具」 |
| 开源 | `/tag/open-source` | 「开源 AI 工具」 |
| 中文界面 | `/tag/chinese-ui` | 「有中文界面的 AI 工具」 |
| API 可用 | `/tag/api-available` | 「AI API 工具」 |
| 无需注册 | `/tag/no-signup` | 「免登录 AI 工具」 |

### 8.5 内链策略

所有页面形成完整内链网络，百度/Google 都更喜欢内链丰富的站点：

```
首页
  ├─→ 分类页 → 工具详情页 → 替代品页
  │                        ↘ 对比页
  ├─→ 场景落地页 → 工具详情页
  ├─→ GitHub 趋势页 → 仓库详情页 → 工具详情页（联动）
  └─→ 资讯页 → 资讯详情 → 关联工具详情页
```

## 十、评论 / 讨论功能

> **决策：暂缓，等工具库 100+ 条、日活稳定后再上线。** 以下为设计预案。

### 10.1 价值评估

对用户：看真实反馈帮助决策（「这个工具国内能用吗」），有地方分享使用技巧。
对运营：UGC = 免费内容，持续丰富页面提升 SEO，评论区活跃度是社区健康的直观体现。
对 SEO：用户评论增加页面文字量和关键词覆盖，结构化数据可加 `Review`/`AggregateRating`。

### 10.2 数据库设计（预留）

```sql
CREATE TABLE comments (
  id          serial PRIMARY KEY,
  target_type text NOT NULL,       -- 'tool' | 'repo' | 'article'
  target_id   text NOT NULL,
  parent_id   integer REFERENCES comments(id),
  author_name text NOT NULL,
  author_email text,               -- 用于 Gravatar，不公开
  author_ip   inet NOT NULL,       -- 合规记录，保存 60 天
  content     text NOT NULL,
  tag         text,                -- '使用体验'|'国内可用性'|'价格'|'使用技巧'
  rating      smallint,            -- 1-5，仅顶级评论
  status      text NOT NULL DEFAULT 'pending',  -- pending|approved|rejected|spam
  upvotes     integer DEFAULT 0,
  created_at  timestamp DEFAULT now()
);
CREATE INDEX ON comments (target_type, target_id, status);
```

### 10.3 上线条件与路径

上线条件：工具库 ≥ 100 条 AND 日活 ≥ 100 UV AND 有人工审核精力。

分三步：
1. **MVP**：匿名评论（填昵称）→ 人工审核 → 发布
2. **自动化**：关键词过滤 + DeepSeek 内容审核
3. **社区化**：接入微信登录，积分体系，老用户免审

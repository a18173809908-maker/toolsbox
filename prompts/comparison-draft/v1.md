## 输出格式（严格 JSON）

```json
{
  "title": "工具A vs 工具B：<核心差异一句话>",
  "verdictOneLiner": "一句话结论（≤40字，明确说谁该选哪个）",
  "summary": "导语段落（≤120字）",
  "sections": [
    { "heading": "核心功能差异", "body": "..." },
    { "heading": "国内可用性", "body": "..." },
    { "heading": "定价与性价比", "body": "..." },
    { "heading": "适用场景", "body": "..." },
    { "heading": "编辑结论", "body": "..." }
  ],
  "whoShouldPickA": ["角色描述1", "角色描述2"],
  "whoShouldPickB": ["角色描述1", "角色描述2"],
  "seoKeywords": ["关键词1", "关键词2", "关键词3"],
  "cellWinners": {
    "价格": "<toolId of cheaper tool, or 'tie'>",
    "国内访问": "<toolId of more accessible tool, or 'tie'>",
    "中文界面": "<toolId, or 'tie'>",
    "免费额度": "<toolId with better free tier, or 'tie', or omit if both have none>",
    "API 可用": "<toolId, or 'tie'>",
    "开源": "<toolId, or 'tie'>",
    "注册门槛": "<toolId with lower barrier, or 'tie'>"
  }
}
```

`cellWinners` 填写规则：
- 值 = toolId（必须与输入数据中的 id 完全一致）
- 值 = `"tie"` 表示两者基本持平
- 某维度无法判断（如双方均无开源）时，完全省略该 key
- 不要强行分胜负：如价格差异 < 20% 可写 `"tie"`
